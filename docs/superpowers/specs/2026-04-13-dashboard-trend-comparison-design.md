# Dashboard 环比趋势设计规范

## 目标

在不增加认知负担、不破坏现有视觉结构的前提下，为「阶段聚焦」模块的 4 张指标卡片增加环比趋势信息，用于表达当前周期相对上一周期的上涨、下跌、持平或不可比状态。

当前模块特征：
- 4 张数据卡片
- 支持切换 `本周 / 本月 / 本年`
- 当前卡片结构稳定，不希望因为新增趋势信息而改变卡片骨架

本次设计的目标是：
- 用户可在 **1 秒内** 感知趋势方向
- 当前主数值仍是绝对主角
- 趋势信息默认轻量、低干扰、适合长期高频使用
- 方案可扩展到 tooltip、dark mode、未来 sparkline

---

## 最终选型

### 推荐方案

采用 **方案 A + A1**：

- **方案 A**：主数值下方增加一行轻量趋势信息
- **A1**：默认只显示 **百分比**
- 卡片右下角叠加一枚 **融入背景的 trend curve arrow**
- 绝对变化量与上一周期值不默认展示，放入 tooltip

### 设计原则

1. **当前值是主，趋势是辅**
2. **方向靠形状 + 正负号 + 文案，不依赖强色彩**
3. **背景箭头属于材质氛围层，不应成为视觉主角**
4. **新增信息只能占用一行，不改变当前卡片扫描路径**

---

## 信息架构

### 卡片结构

每张卡片统一采用以下层级：

```text
Card
├─ Label
├─ Primary Value + Unit
├─ Trend Row
│  ├─ Trend Percent / 持平 / 暂无环比
│  └─ Comparison Label（vs 上周 / 上月 / 上年）
└─ Background Trend Arrow（SVG，置于内容层下）
```

### 推荐阅读顺序

`指标标题 → 当前值 → 环比趋势`

这使新增趋势信息自然成为主数值的辅助说明，而不是独立信息块。

### 趋势信息位置

趋势信息放在 **主数值下方，左对齐主数值**。

不采用以下位置：

- **标题旁**：与主值距离远，容易被理解成状态标签
- **主值右侧**：容易与主值争夺视觉重心
- **卡片顶部 Badge**：会给高密度 dashboard 增加额外视觉节点

---

## 数据展示规则

### 默认展示内容

默认只显示百分比：

- `+8.2% vs 上周`
- `-4.3% vs 上月`
- `持平`
- `暂无环比`

### 不默认展示绝对值的原因

- 4 卡并排时会提高拥挤感
- 不同指标单位不同，绝对值不利于横向快速比较
- 百分比更适合表达“变化强弱”

### 绝对值的去处

通过 tooltip 展示：

- 当前值
- 上一周期值
- 环比百分比
- 绝对变化量

示例：

- 当前：82.4 km
- 上周：76.2 km
- 环比：+8.2%
- 变化量：+6.2 km

---

## 视觉方案对比与结论

### 方案 A：辅助行 + 背景趋势曲线箭头（采用）

形式：
- 主值下方一行趋势信息
- 卡片右下角一枚低对比曲线箭头

优点：
- 不破坏现有结构
- 扫描路径稳定
- 最适合 4 卡并排的高信息密度场景
- 具备长期使用的耐看性

缺点：
- 装饰性不如 badge 强
- 背景箭头需要严格控制权重

### 方案 B：顶部 Badge 标签

形式：
- 右上角 badge 显示变化百分比
- 主值下方只显示对比周期

不采用原因：
- 视觉节点增加过多
- 多卡并排时容易显得热闹
- 更偏产品化标签，不够克制

### 方案 C：右侧大面积装饰趋势箭头

形式：
- 右侧大趋势箭头
- 文案极简

不采用原因：
- 最容易喧宾夺主
- 不适合当前高密度 dashboard
- 对长期阅读耐受性较差

---

## 趋势图形规范

### 图形形式

采用 **trend curve arrow**，要求如下：

#### 上涨
- 从左侧开始为平缓或微升线段
- 向右逐渐抬升、弯曲
- 末端以箭头收尾

#### 下跌
- 从左侧开始为平缓或微降线段
- 向右逐渐下弯
- 末端以箭头收尾

#### 持平
- 近似水平线段
- 末端可保留很弱的箭头收尾

### 图形语义

方向主要通过 **曲线走势** 传达，而不是依赖颜色。

---

## 背景箭头布局规范

### 放置位置

- 位于卡片 **右下角**
- 置于内容层下方
- 不得穿过主数值主体区域

### 尺寸建议

以桌面卡片为基准：

- 宽度：卡片宽度的 **40%–46%**
- 高度：卡片高度的 **28%–34%**
- 距右：**12–16 px**
- 距下：**10–14 px**

### 线条建议

- 线宽：**1.5–2 px**
- 箭头头部长度：**6–8 px**
- 以线性路径为主，不使用实心大面积填充

### 透明度建议

- 深色模式：**10%–16%**
- 浅色模式：**8%–12%**

### 禁止项

- 不允许使用高饱和红色或绿色大面积背景
- 不允许发光 halo
- 不允许阴影过重
- 不允许尺寸大到切入主值阅读区域

---

## 颜色策略

### 总体原则

使用 **低饱和中性色方案**。  
颜色只作为弱提示，不承担唯一语义。

### 背景箭头色

#### Up
- 建议：Muted Blue / Mist Cyan
- 示例：`rgba(140, 180, 255, 0.16)`

#### Down
- 建议：Muted Slate / Cool Gray
- 示例：`rgba(170, 180, 195, 0.14)`

#### Flat
- 建议：Neutral Gray
- 示例：`rgba(150, 156, 168, 0.12)`

### 趋势文案色

#### Up
- `rgba(210, 225, 255, 0.86)`

#### Down
- `rgba(220, 224, 232, 0.76)`

#### Flat
- `rgba(210, 210, 210, 0.68)`

#### Comparison Label
- `rgba(255, 255, 255, 0.42)`

### 不采用的方案

- 高饱和红涨绿跌
- 纯金融语义色作为唯一判断依据
- 强烈 cyan / magenta 发光色

---

## 状态规范

### 1. 上涨（+）

展示：
- 背景：上扬趋势曲线箭头
- 文案：`+8.2%`
- 辅助文案：`vs 上周 / 上月 / 上年`

组合示例：
- `+8.2% vs 上周`

### 2. 下跌（-）

展示：
- 背景：下弯趋势曲线箭头
- 文案：`-4.3%`
- 辅助文案：`vs 上周 / 上月 / 上年`

组合示例：
- `-4.3% vs 上月`

### 3. 无变化（0）

展示：
- 背景：水平趋势箭头
- 文案：`持平`
- 可不再重复显示 `0.0%`

原因：
- 更快理解
- 更少机械感
- 语言更自然

### 4. 数据缺失（N/A）

展示：
- 不显示背景箭头
- 文案：`暂无环比`
- tooltip 补充：`缺少上周期数据，无法计算环比`

---

## 边界情况

### 上一周期 = 0，本周期 > 0

不显示 `∞%`。

建议展示：
- `新增`

tooltip 补充：
- `上周期为 0，百分比不适用`

### 上一周期 > 0，本周期 = 0

允许显示：
- `-100%`

### 极小波动

建议设定阈值：

- `|变化| < 0.5%` → 视为 `持平`

目的是减少无意义抖动，保持卡片稳定性。

---

## 时间周期映射规则

趋势信息必须与当前筛选周期同步变化：

- `本周` → `vs 上周`
- `本月` → `vs 上月`
- `本年` → `vs 上年`

该映射必须作为固定规则写入组件逻辑，避免硬编码错误。

---

## 视觉层级

### 主辅关系

1. **当前值**：一级信息
2. **环比信息**：二级信息
3. **背景箭头**：三级氛围层

### 桌面端建议尺寸

- 卡片内边距：**20–24 px**
- Label：**11–12 px**
- 主值：**28–32 px**
- 单位：**14–16 px**
- 趋势行：**12–13 px**

### 移动 / 紧凑状态

- 主值：**24–28 px**
- 趋势行：**11–12 px**

### 间距建议

- Label → 主值：**14–18 px**
- 主值 → 趋势行：**8–10 px**
- 趋势行 → 卡片底部：**0–2 px**
- 背景箭头与正文最小安全距：**12 px**

### 透明度建议

- 主值：`white / 92%`
- 单位：`white / 52%`
- Label：`white / 46%`
- 趋势主文案：`white / 72%`
- `vs 上周 / 上月 / 上年`：`white / 42%`

---

## 动效规范

### 总原则

允许存在微动效，但必须轻量且服务于状态更新。

### 周期切换时

- 主值：数字过渡
- 趋势文案：`fade + 4px 上移`
- 背景箭头：仅做 `opacity 0 → 1`

### 动效时长

- **160–220ms**

### 禁止项

- 不允许呼吸灯
- 不允许浮动
- 不允许循环路径描边
- 不允许大面积颜色闪烁

---

## Tooltip 规范

### 触发方式

- Hover / Focus 触发
- 不默认常驻

### 信息结构

- 当前值
- 上一周期值
- 环比百分比
- 绝对变化量
- 必要时补充计算口径

### 示例

```text
当前：82.4 km
上周：76.2 km
环比：+8.2%
变化量：+6.2 km
```

---

## 可用性与认知要求

### 1 秒理解测试

用户应能在不阅读 tooltip 的情况下，在 1 秒内判断：

- 当前指标是升还是降
- 对比对象是谁
- 变化是否显著

为满足这一点，组件必须同时具备：

- 明确的正负号 / 持平词汇
- 明确的 `vs 上周 / 上月 / 上年`
- 明确的曲线方向

### 风险控制

#### 颜色文化差异

不能只靠颜色判断涨跌，必须保留正负号与方向曲线。

#### 文案歧义

禁止只显示 `+8.2%` 而没有 `vs xxx`。

#### 过度装饰

背景趋势箭头必须始终服务于卡片氛围，不得成为主要视觉元素。

---

## Dark Mode 适配

当前 OarBoard 已采用深色玻璃态，此方案天然适配 dark mode。

### 深色模式重点

- 箭头更像材质纹理的一部分
- 小字对比度不能过低
- 趋势状态不通过高饱和色强调

### 如果未来扩展浅色模式

- 箭头透明度再降低一级
- 趋势文案改用更深的中性灰
- 保持同样的信息层级，不增加饱和色

---

## 未来扩展方向

### 1. Sparkline

当前不默认启用。  
未来可以扩展为：

- 默认：静态 trend curve arrow
- hover / expanded：显示迷你 sparkline

这样既保留简洁性，又能提供更深入趋势分析。

### 2. 微交互

可补充：

- hover 时卡片整体亮度提升不超过 4%
- 趋势文案 opacity 轻微增加
- tooltip 平滑出现

---

## 开发实现建议

### 组件职责

建议将趋势信息拆成独立可复用子结构：

- `TrendMeta`
  - 接收状态：`up | down | flat | na | new`
  - 接收 percent label
  - 接收 comparison label
- `TrendArrowBackground`
  - 接收状态并渲染对应 SVG path
  - 统一管理位置、透明度、颜色和 pointer-events

### 逻辑映射建议

```text
deltaPercent > 0.5   => up
deltaPercent < -0.5  => down
abs(deltaPercent) < 0.5 => flat
previousMissing      => na
previousZero && currentPositive => new
```

### 文案映射建议

- `up`：`+8.2% vs 上周`
- `down`：`-4.3% vs 上月`
- `flat`：`持平`
- `na`：`暂无环比`
- `new`：`新增`

---

## 验收标准

设计落地成功需满足以下条件：

1. 当前主数值仍然是卡片第一视觉焦点
2. 趋势信息新增后，卡片高度与整体节奏不失衡
3. 用户能快速区分上涨、下跌、持平、暂无环比
4. 背景箭头不会抢占主值阅读
5. 周/月/年切换后，对比文案同步正确
6. 高密度场景下 4 卡并排依然保持整洁
7. 深色模式下趋势箭头融入背景而非变成装饰主角

---

## 结论

本次设计采用 **克制、长期可用、可扩展** 的方向：

- 布局上，趋势信息作为主值下方的一行辅助说明
- 表现上，使用融入背景的趋势曲线箭头
- 内容上，默认仅显示百分比
- 交互上，通过 tooltip 补充更深层数据

该方案最适合当前 OarBoard 的深色玻璃态 dashboard，也最符合高频使用产品对稳定性、识别效率与视觉耐受性的要求。
