# 🚣‍♂️ OarBoard 项目开发指导手册

## 1. 项目愿景
**OarBoard** 是一个专为个人设计的划船机运动数据可视化面板。它通过接入“摩刻健身”的接口数据，以 **Apple Fitness Rings (健身圆环)** 的美学风格，提供比原生 App 更直观、更美观的数据分析体验。

## 2. 技术栈要求
*   **框架**: Next.js 16 (App Router)
*   **样式**: Tailwind CSS (需支持响应式、暗黑模式、毛玻璃效果)
*   **组件库**: Shadcn UI
*   **图标**: Lucide React
*   **动画**: Framer Motion
*   **图表**: Recharts (用于展示速度与桨频折线图)
*   **部署**: Vercel

---

## 3. 分阶段 Prompt 指令

### 第一阶段：数据解析与逻辑建模
**按照以下指令进行：**

> 我正在开发一个名为 **OarBoard** 的划船机数据看板。`mokfiness.xml` 是我从“摩刻健身”App 抓包获取的接口文件。
>
> **请执行以下任务：**
>
> 1. **结构分析**：详细解析 XML 中的 JSON 数据，识别出以下关键接口：
>    - 运动历史列表接口 (Workout List)
>    - 运动详情接口 (Workout Detail)
>    - 个人总计数据接口 (User Totals)
> 2. **类型定义**：使用 TypeScript 定义上述接口的 Data Models。
> 3. **单位转换逻辑**：编写工具函数，将原始数据转换为人类可读格式：
>    - 时长：秒 -> `HH:mm:ss`
>    - 距离：米 -> `km` (保留两位小数)
>    - 配速：将速度转换为划船机特有的 `500米配速` (例如 02:15/500m)
>    - 日期：时间戳 -> `YYYY-MM-DD HH:mm`

---

### 第二阶段：后端代理与安全
**按照以下指令进行：**

> 我需要解决接口跨域问题并保护我的 Token。请基于 Next.js API Routes 实现以下功能：
>
> 1. **API Proxy**：创建一个后端代理路由 `/api/moke/[...path]`，将请求转发给摩刻真实的 API 地址。
> 2. **鉴权处理**：从环境变量 `MOKE_AUTHORIZATION` 中读取 Token，并自动注入到转发请求的 Header 中。
> 3. **模拟数据**：如果环境变量不存在，请返回我在 XML 中提供的测试假数据，以便开发环境调试。

### 第三阶段：构建 Apple 风格健身圆环 (核心组件)
**按照以下指令进行：**

> 请作为高级前端工程师，使用 Tailwind CSS 和 SVG 开发一个 **Apple Fitness 风格的健身圆环组件**：
>
> 1. **视觉设计**：
>    - 三个层叠圆环：外环红色（热量消耗）、中环绿色（运动时长）、内环蓝色（运动距离/桨次）。
>    - 具有圆角末端、渐变色背景以及背后的半透明阴影轨道。
>    - 包含进入页面的旋转动画效果。
> 2. **交互**：圆环中心显示当天的核心指标（如“总运动时长”）。
> 3. **响应式**：在移动端和 PC 端都能完美显示。

### 第四阶段：主仪表盘与详细图表
**按照以下指令进行：**

> 现在构建 OarBoard 的主界面：
>
> 1. **顶部总计卡片**：展示总运动时长、总消耗、总运动距离（毛玻璃效果卡片）。
> 2. **每日历史列表**：使用卡片流形式展示每一天的运动概要。
> 3. **运动详情模态框**：点击某次运动，弹窗显示：
>    - 核心指标：平均配速、平均速度、平均桨频、总桨次。
>    - **多维度折线图**：使用 Recharts，横轴为时间（分钟），纵轴双 Y 轴分别展示“速度”和“桨频”的波动趋势。
>    - 配色方案：深色背景，图表线条使用鲜艳的荧光色。

## 4. 关键交互细节（给 AI 的补充建议）
*   **毛玻璃效果**: 强制使用 `backdrop-blur-md bg-white/10` 样式，营造高端感。
*   **动画**: 使用 `framer-motion` 为页面过渡、卡片加载、卡片 hover 动效、图表加载动画和圆环伸展等添加平滑动画。
*   **单位规范**: 划船机运动中最看重 **500m Pace**（配速）和 **SPM**（每分钟桨频），如果接口数据中有这些数据，请确保这两项在 UI 中最显眼。

## 5. 部署配置
1.  在 Vercel 环境变量配置:
    *   `MOKE_AUTHORIZATION`: 填入你抓包获取的 Token。
    *   `NEXT_PUBLIC_BASE_URL`: 摩刻 API 的 Base URL。
